Welcome to 3rd Challenge of this Dev Weekend! 🚩  For this challenge, You have been reassigned as a data interception analyst. A rogue employee was caught attempting to smuggle highly classified data out of the internal network. Security agents intercepted their outgoing communications, but all they found was a seemingly innocent, boring text file (classified_memo.txt).  However, our intelligence team noticed the file size is slightly larger than it should be. The employee used Steganography to hide a secret payload within the text itself.  The Vulnerability - Invisible Data & The Double-Wrap:: The Unicode standard contains special formatting characters known as "Zero-Width" characters. To a human reading the file, these characters are completely invisible. To a computer reading the raw bytes, they are explicitly there. The rogue employee encoded a secret flag into binary (0s and 1s) and mapped them to invisible characters:  Zero-Width Space (\u200b) represents a binary 0 Zero-Width Non-Joiner (\u200c) represents a binary 1 These invisible characters were injected directly into the memo. To make matters worse, real threat actors rarely use just one layer of obfuscation. The data hidden inside this invisible binary sequence is actually encoded in Base64.  Project Structure: You are given a few key files:  classified_memo.txt: The seemingly innocent text file containing the hidden data. exploit.py: Your main extraction script. Your Task Your objective is to write a Python script that reads the text file, extracts the invisible zero-width characters, translates them back into an ASCII string, and decodes the Base64 layer to reveal the final flag.  Implementation Objectives Open exploit.py in your code editor.  TODO 1 (Extract the Binary): Read the contents of classified_memo.txt. Loop through every character in the text. If the character is \u200b, append a "0" to your binary string. If the character is \u200c, append a "1" to your binary string. TODO 2 (Decode the ASCII): Computers read text in 8-bit chunks (bytes). Loop through your extracted binary string, grabbing 8 characters at a time. Convert that 8-character binary string into a base-2 integer, and then convert that integer back into an ASCII character. Docs: Python chr() & int()  TODO 3 (The Base64 Double-Wrap): The ASCII string you just recovered is not the final flag - it is an encoded string. Remember to decode the resulting bytes back into a standard utf-8 string. Docs: Python base64  Finally, run python exploit.py in your terminal (or click the RUN button).  Expected Output: Once completed, your script successfully parses the document, cracks both layers of obfuscation, and prints the hidden flag.
